# 角色卡格式规范

## 推荐格式：YAML

YAML 比 JSON 省 token，可读性更好。引擎同时支持两种格式。

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✓ | 角色名 |
| age | 推荐 | 年龄+性别，如 "18岁，女" |
| description | ✓ | 外貌描写（200-400字） |
| personality | ✓ | 性格描写（表层→深层→矛盾点） |
| behavior | 推荐 | 具体可观察的行为习惯 |
| speech | ✓ | 语气/口癖/说话方式 |
| likes | 可选 | 喜好 |
| dislikes | 可选 | 厌恶 |
| backstory | 推荐 | 背景故事 |
| scenario | ✓ | 当前场景设定 |
| first_mes | ✓ | 第一条消息（**最重要**） |
| mes_example | 推荐 | 对话范例（3-5组） |
| system_prompt | 推荐 | 心理建模规则/额外约束 |

## 模板

```yaml
name: 角色名
age: "年龄，性别"

description: |
  # 外貌描写（200-400字）
  # 要求：具体到能画出来的程度
  # 结构：发型颜色 → 眼睛 → 肤色体型 → 身高 → 日常穿着 → 标志性配饰

personality: |
  # 性格描写（300-500字）
  # 结构：表层性格 → 深层性格 → 矛盾点 → 对{{user}}的态度
  # 关键：写出"表里不一"——表面X，实则Y

behavior: |
  # 行为模式（100-200字）
  # 写具体的、可观察的行为
  # ✅ "紧张时下意识摸耳垂"
  # ❌ "有时候会紧张"

speech: |
  # 语气/口癖（100-200字）
  # 写出这个角色说话和别人的区别
  # 包含：语速、音量、口头禅、称呼习惯、情绪变化时的语言变化

likes: |
  喜好（用顿号分隔）

dislikes: |
  厌恶

backstory: |
  # 背景故事（200-400字）
  # 包含成长弧和关键转折点

scenario: |
  # 当前场景设定（100-200字）
  # 用 {{user}} 和 {{char}} 占位

first_mes: |
  # 第一条消息（200-500字）
  # 这是最重要的字段！决定后续所有回复的文风
  # 用 *星号* 写动作，用 "引号" 写对话
  # 必须体现：台词风格 + 性格核心 + 场景氛围

mes_example: |
  {{user}}: 示例输入
  {{char}}: *动作描写* "对话内容"

system_prompt: |
  # 心理建模规则（可选但推荐）
  # 依恋模式：[类型] — 行为预测
  # 防御机制：[机制] — 触发场景
  # 核心图式：[图式] — 解读滤镜
  # 需求层级：[层级] — 触发话题
  # 道德推理：[类型] — 判断逻辑
```

## 占位符

- `{{user}}` — 用户/玩家
- `{{char}}` — 当前角色

## JSON 格式

引擎也支持 SillyTavern 标准 JSON 格式（spec: chara_card_v2）。
PNG 角色卡可通过 `png_parser.py` 导入。
